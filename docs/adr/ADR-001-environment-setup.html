ADR-001 — Environment Setup

Accepted June 2026

Starting a greenfield insurance claims application. Needed a development environment that works on a locked-down corporate laptop (no admin rights, restricted network) and from home, without duplicating setup across machines or relying on local installs.

01

Decisions

Development environment
GitHub Codespaces
Corporate laptop has no admin rights and may block local Docker installs. Codespaces runs entirely in the browser — nothing to install, nothing to configure per machine. One cloud VM accessible from anywhere, same state each time.
Rejected Local Docker Desktop — blocked by corporate policy; requires admin rights.
Database
devcontainer Postgres
Runs as a sidecar container alongside the app — no internet required, starts automatically with the environment, zero external accounts to manage. Data persists in a Docker volume across container restarts.
Rejected Neon (managed cloud DB) — introduces a network dependency and external account. Remains a fallback if Codespaces storage becomes a constraint.
Python version
Python 3.12
Current stable release at project start. No legacy constraints pushing toward an older version. FastAPI and SQLAlchemy both support it fully. Gets the latest typing and performance improvements.
Rejected Python 3.11 or below — no reason to accept an older runtime for a greenfield project.
Web framework
FastAPI
Generates OpenAPI docs automatically (Swagger UI at /docs), native async support, and uses Python type hints for request/response validation — keeping the codebase explicit and self-documenting from the start.
Rejected Flask — no built-in validation or docs generation; requires more manual wiring. Django — too heavyweight for an API-first app with no CMS or admin panel need.
02

Consequences

Gain

Zero setup on any machine. Corporate laptop, home desktop, or any browser — same environment, same state, no configuration drift.

Gain

The devcontainer is the environment spec. Anyone who opens the repo gets an identical machine — reproducible by design, not by convention.

Watch

GitHub dependency. If GitHub is unavailable, work stops. Accepted tradeoff given the corporate laptop constraint — no viable local alternative.

Watch

Postgres data is ephemeral if the codespace is deleted (not stopped). Data lives in a Docker volume — migrations introduced in Slice 1 will make the schema recoverable from code, so only seed data would be lost.

Note

Free tier gives ~60 active hours/month on a 2-core machine. Sufficient for current pace. Revisit if development velocity increases significantly or compute-heavy tasks are introduced.
